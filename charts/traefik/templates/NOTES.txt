Traefik has been deployed with automatic Let's Encrypt TLS!

1. Check the external IP assigned to the LoadBalancer service:
   kubectl -n {{ .Release.Namespace }} get svc -l app.kubernetes.io/name=traefik

2. Point your domain(s) to the IP above (or your router's public IP with port forwarding).

3. Let's Encrypt certificates are stored in:
   PVC traefik-data (path: /data/acme.json)

4. TLS certificate resolver name: letsencrypt
   Use this name in IngressRoute resources:
     tls:
       certResolver: letsencrypt

5. The dashboard is exposed at https://traefik.home.agu.com.ar behind HTTP
   basic auth. Create the credentials secret before/just after first sync:
     htpasswd -nb admin 'your-password' | \
       kubectl create secret generic traefik-dashboard-auth \
         -n {{ .Release.Namespace }} --from-file=users=/dev/stdin
   (or set dashboardAuth.users in values.yaml for a throwaway password).

NOTE: The HTTP-01 challenge requires that port 80 be reachable from the
internet. For a home lab, configure port forwarding on your router for
ports 80 and 443 to your RPi's local IP.

Happy routing! 🌐

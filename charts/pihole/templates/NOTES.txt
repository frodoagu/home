2. Admin password:
{{- if .Values.admin.existingSecret }}
   From secret "{{ .Values.admin.existingSecret }}" (key {{ .Values.admin.existingSecretKey }}).
{{- else }}
   No secret configured — Pi-hole auto-generated a random password. Read it with:
     kubectl -n {{ .Release.Namespace }} logs deploy/{{ include "pihole.fullname" . }} | grep -i password
   or set your own:
     kubectl -n {{ .Release.Namespace }} exec deploy/{{ include "pihole.fullname" . }} -- pihole setpassword
   Recommended: create a secret and set admin.existingSecret:
     kubectl create secret generic pihole-admin -n {{ .Release.Namespace }} --from-literal=password=CHANGEME
{{- end }}

   {{ .Values.dhcp.start }} – {{ .Values.dhcp.end }} (gateway {{ .Values.dhcp.router }}) and

   ⚠️  TURN OFF the DHCP server on your router first — two DHCP servers on one
       LAN will fight and break connectivity. Then reboot devices (or wait for
       lease renewal) so they pick up the new DNS.

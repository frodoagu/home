1. Web admin UI (Google sign-in gated):
   https://{{ .Values.ingress.host }}/admin
   (a DNS A record for {{ .Values.ingress.host }} must point at your RPi, like the
    other agu.com.ar services)

2. Admin login:
{{- if .Values.admin.existingSecret }}
   Password from secret "{{ .Values.admin.existingSecret }}" (key {{ .Values.admin.existingSecretKey }}).
{{- else if .Values.admin.disablePassword }}
   Pi-hole's own login is DISABLED — google-auth gates the UI. (Required: the
   google-auth-signin errors middleware would otherwise hijack the API's 401 and
   the SPA would report "Server unreachable!".) Set admin.disablePassword: false
   + admin.existingSecret to use a password instead.
{{- else }}
   No secret configured — Pi-hole auto-generated a random password. Read it with:
     kubectl -n {{ .Release.Namespace }} logs deploy/{{ include "pihole.fullname" . }} | grep -i password
{{- end }}

{{- if .Values.dhcp.enabled }}

3. DHCP is ENABLED. Pi-hole hands out IPs in the range
   {{ .Values.dhcp.start }} – {{ .Values.dhcp.end }} (gateway {{ .Values.dhcp.router }}) and
   advertises ITSELF as the DNS server.

   ⚠️  TURN OFF the DHCP server on your router first — two DHCP servers on one
       LAN will fight and break connectivity. Then reboot devices (or wait for
       lease renewal) so they pick up the new DNS.
